# FrugaLLM 2.0 — Configuration Reference

## Configuration Files

| File | Purpose | Auto-Generated? |
|------|---------|-----------------|
| `config/litellm_config.yaml` | Main LiteLLM proxy configuration | No — edit freely |
| `config/dynamic_models.yaml` | Dynamic model roster from OpenRouter | Yes — by sidecar |
| `.env` | API keys and runtime configuration | No — edit freely |

## Environment Variables

### Required

| Variable | Description |
|----------|-------------|
| `OPENROUTER_API_KEY` | OpenRouter API key for model inference. Get one at [openrouter.ai/keys](https://openrouter.ai/keys) |

### FrugaLLM Settings

| Variable | Default | Description |
|----------|---------|-------------|
| `FRUGALLM_PROXY_PORT` | `4000` | Port for the LiteLLM proxy |
| `FRUGALLM_PROXY_URL` | `http://127.0.0.1:4000` | Full proxy URL (used by CLI) |
| `FRUGALLM_MASTER_KEY` | `sk-frugallm-master` | Authentication key for the proxy |
| `FRUGALLM_CONFIG_DIR` | `./config` | Directory containing YAML configs |
| `FRUGALLM_POLL_INTERVAL` | `300` | Sidecar poll interval in seconds |

### Local Model Settings

| Variable | Default | Description |
|----------|---------|-------------|
| `FRUGALLM_LOCAL_MODEL` | `llama3.2:latest` | Ollama model name for local fallback |
| `FRUGALLM_LOCAL_URL` | `http://127.0.0.1:11434` | Ollama base URL |

### GPU Node Settings (Optional)

| Variable | Default | Description |
|----------|---------|-------------|
| `FRUGALLM_GPU_MODEL` | *(empty)* | Model path on GPU node (e.g., `/models/llama.gguf`) |
| `FRUGALLM_GPU_URL` | *(empty)* | GPU node base URL (e.g., `http://100.x.x.x:8080/v1`) |

### Telemetry Settings (Optional)

| Variable | Description |
|----------|-------------|
| `LANGFUSE_SECRET_KEY` | Langfuse secret key for tracing |
| `LANGFUSE_PUBLIC_KEY` | Langfuse public key |
| `LANGFUSE_HOST` | Langfuse host URL (e.g., `https://us.cloud.langfuse.com`) |
| `DATABASE_URL` | PostgreSQL connection string for spend logging |

### API Key Separation (Optional)

| Variable | Description |
|----------|-------------|
| `OPENROUTER_MANAGEMENT_KEY` | Separate key for sidecar model polling (avoids rate limit conflicts) |
| `GEMINI_API_KEY` | Google Gemini key for paid fallback models |

## LiteLLM Config (`config/litellm_config.yaml`)

### Model List

Define static model endpoints under `model_list`:

```yaml
model_list:
  - model_name: auto          # The alias clients use
    litellm_params:
      model: ollama/llama3.2:latest   # Actual model identifier
      api_base: http://127.0.0.1:11434
      timeout: 60
      max_retries: 2
```

### Router Settings

Control retry behavior and routing strategy:

```yaml
router_settings:
  routing_strategy: simple-shuffle    # or: latency-based-routing, usage-based-routing
  num_retries: 2                      # Total retry attempts
  timeout: 300                        # Default request timeout (seconds)
  stream_timeout: 300                 # Streaming request timeout
  retry_after: 60                     # Seconds to wait before retrying a failed model
  allowed_fails: 3                    # Failures before a model is temporarily removed
```

### General Settings

```yaml
general_settings:
  master_key: sk-frugallm-master    # Auth key — CHANGE THIS!
  alerting: []                       # Optional: ["slack", "email"]
```

### LiteLLM Settings

```yaml
litellm_settings:
  # Telemetry callbacks (uncomment to enable)
  # success_callback: ["langfuse", "prometheus"]
  # failure_callback: ["langfuse", "prometheus"]

  # Response caching
  cache: true
  cache_params:
    type: local          # In-memory cache
    ttl: 300             # Cache TTL in seconds

  # Compatibility
  drop_params: true      # Drop unsupported params (important for local models)
  set_verbose: true      # Verbose logging

  # Custom middleware
  callbacks: frugallm.custom_callbacks.frugallm_proxy_handler
```

### Include Files

The `include` directive merges another YAML file into the config:

```yaml
include:
  - dynamic_models.yaml    # Auto-generated by sidecar
```

## Dynamic Models (`config/dynamic_models.yaml`)

This file is auto-generated by the sidecar every 5 minutes. It contains:

1. **Balanced Chain**: Free general-purpose models sorted by recency
2. **Reasoning Chain**: Free reasoning-capable models
3. **Fallback definitions**: Linear chains from each model to the next

You should **not** edit this file manually. To customize:
- Adjust the sidecar's selection criteria in `frugallm/dynamic_roster_sidecar.py`
- Set `FRUGALLM_LOCAL_MODEL` and `FRUGALLM_GPU_*` for fallback endpoints

## Middleware Configuration

The middleware hooks are defined in `frugallm/custom_callbacks.py` and registered via:

```yaml
litellm_settings:
  callbacks: frugallm.custom_callbacks.frugallm_proxy_handler
```

To disable all middleware, remove or comment out the `callbacks` line.

To customize individual middlewares, subclass `FrugaLLMProxyHandler` and override the methods you want to change.
